## 📌 Git 컨벤션

### Branch 전략 (Git Flow)

```
main (프로덕션)
  ↑
release (배포 준비)
  ↑
develop (개발 통합)
  ↑
feature/* (기능 개발)
hotfix/*  (긴급 수정)
```

### Branch 명명 규칙
```
feature/기능-이름           예: feature/community-board-crud
feature/BE-게시글-CRUD     예: feature/BE-post-crud
feature/FE-채팅-UI         예: feature/FE-chat-ui

hotfix/버그-설명           예: hotfix/chat-timeout-error

docs/문서-내용             예: docs/api-specification
```

### Branch 관리 실행 예시
```bash
# 1. develop 브랜치 기반 feature 생성
git checkout develop
git pull origin develop
git checkout -b feature/community-board-crud

# 2. 작업 후 커밋
git add .
git commit -m "feat: 게시판 CRUD API 구현"

# 3. Push 및 Pull Request
git push origin feature/community-board-crud
# GitHub에서 Pull Request 생성 → Review → Merge
```

---

### Commit 메시지 컨벤션

**형식**: `type: 주제`

**Type 분류**
```
feat:      새로운 기능 추가
fix:       버그 수정
docs:      문서 수정/추가 (코드 변경 없음)
style:     코드 스타일 변경 (기능 변경 없음, 포맷팅, 세미콜론 등)
refactor:  코드 리팩토링 (기능 변경 없음)
test:      테스트 코드 추가/수정
chore:     빌드, 의존성, 환경 설정 등
```

**예시 (한 줄 커밋)**
```
feat: 게시글 CRUD API 구현
feat: 챗봇 /api/chat 엔드포인트 구현
fix: 게시글 수정 시 비밀번호 검증 버그 수정
docs: API 명세서 업데이트
refactor: PostService 클래스 구조 개선
test: Post 모델 단위 테스트 추가
chore: requirements.txt 업데이트 (FastAPI 0.104.0)
```

**상세 커밋 (선택사항)**
```
feat: 게시글 CRUD API 구현

- POST /api/posts: 게시글 작성 (비밀번호 검증)
- GET /api/posts: 목록 조회 (페이지네이션)
- PUT /api/posts/{id}: 게시글 수정
- DELETE /api/posts/{id}: 게시글 삭제

각 엔드포인트에 Pydantic 스키마 검증 추가
```
